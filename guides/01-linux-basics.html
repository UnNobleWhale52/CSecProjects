Guide 01 - Linux Fundamentals

Last updated: January 2026 | Reading time: ~15 minutes

üìñ What is Linux?

Linux is a free, open-source operating system kernel that was first released in 1991 by Linus Torvalds. Unlike proprietary operating systems like Windows or macOS, Linux's source code is freely available for anyone to view, modify, and distribute.

When most people say "Linux," they're actually referring to a complete operating system that includes the Linux kernel plus a collection of software, tools, and a desktop environment. These complete packages are called Linux distributions (or "distros" for short).

Why Linux Matters for Cybersecurity

Linux is the backbone of cybersecurity for several important reasons:

  • Server Dominance: Over 96% of web servers run on Linux
  • Security Tools: Most security tools are built for Linux first
  • Transparency: Open source means you can audit the code
  • Control: Full access to system internals and configurations
  • Customization: Build exactly what you need, nothing more

üí° Pro Tip

If you're pursuing a career in cybersecurity, becoming proficient in Linux is not optional‚Äîit's essential. Start by using Linux daily, even for simple tasks, to build muscle memory with commands and concepts.

üíª The Terminal

The terminal (also called shell, command line, or console) is your primary interface for interacting with Linux. While graphical interfaces exist, the terminal provides:

  • Faster execution of tasks
  • Access to all system functions
  • Ability to automate with scripts
  • Remote server management via SSH

Understanding the Prompt

When you open a terminal, you'll see a prompt that typically looks like this:

user@hostname:~$

Let's break this down:

  • user - Your username
  • @ - Separator
  • hostname - The computer's name
  • : - Separator
  • ~ - Current directory (~ means home directory)
  • $ - Regular user (changes to # for root)

‚ö†Ô∏è Warning

When the prompt shows # instead of $, you're operating as the root user with full system privileges. Be extremely careful‚Äîcommands executed as root can damage your system!

üîß Essential Linux Commands

Here are the fundamental commands every Linux user should know. Master these before moving to more advanced topics.

Navigation Commands

# Print Working Directory - shows where you are
pwd
/home/user

# List directory contents
ls
Documents  Downloads  Pictures

# List with details (permissions, size, date)
ls -la
drwxr-xr-x 2 user user 4096 Jan  3 10:00 Documents
-rw-r--r-- 1 user user  123 Jan  3 09:30 notes.txt

# Change Directory
cd /var/log      # Go to /var/log
cd ~             # Go to home directory
cd ..           # Go up one directory
cd -             # Go to previous directory

File Operations

# Create an empty file
touch newfile.txt

# Create a directory
mkdir projects
mkdir -p projects/web/html  # Create nested directories

# Copy files
cp file.txt backup.txt          # Copy file
cp -r folder/ backup_folder/    # Copy directory recursively

# Move/rename files
mv oldname.txt newname.txt      # Rename
mv file.txt /tmp/               # Move to /tmp

# Remove files (BE CAREFUL!)
rm file.txt                      # Remove file
rm -r folder/                    # Remove directory
rm -rf folder/                   # Force remove (DANGEROUS)

‚ö†Ô∏è Critical Warning

The command rm -rf / or rm -rf /* will delete your entire system! Always double-check your rm commands before pressing Enter. There is no "trash bin" or "undo" for rm in Linux.

Viewing File Contents

# Display entire file
cat /etc/hostname

# View file with pagination
less /var/log/syslog        # Use arrow keys, q to quit
more /var/log/syslog        # Similar but simpler

# Show first/last lines
head -n 20 file.txt         # First 20 lines
tail -n 20 file.txt         # Last 20 lines
tail -f /var/log/syslog     # Follow file in real-time (great for logs!)

System Information

# Current user
whoami
user

# System information
uname -a
Linux hostname 5.15.0-generic #1 SMP x86_64 GNU/Linux

# Disk usage
df -h

# Memory usage
free -h

# Running processes
ps aux
top            # Interactive process viewer
htop           # Better interactive viewer (if installed)

Getting Help

# Manual pages - comprehensive documentation
man ls

# Quick help
ls --help

# Find what a command does
whatis ls
ls (1) - list directory contents

# Find command location
which python
/usr/bin/python

üí° Pro Tip

The man command is your best friend. Whenever you encounter a new command, read its manual page first. Use / to search within man pages and q to quit.

üìÅ Linux File System Structure

Understanding the Linux file system hierarchy is crucial for navigation and security work. Here are the most important directories:

Directory Purpose
/ Root directory - the top of the file system
/home User home directories
/root Root user's home directory
/etc System configuration files
/var Variable data (logs, databases, mail)
/var/log System and application logs
/bin Essential user binaries
/sbin System binaries (admin commands)
/usr User programs and data
/tmp Temporary files (cleared on reboot)
/opt Optional/third-party software

üîí Security Note

As a security professional, you'll spend a lot of time in /etc (configurations), /var/log (logs), and /home (user artifacts). These directories often contain evidence of attacks or misconfigurations.

üéØ Practice Exercises

Try these exercises to reinforce what you've learned:

  1. Navigation: Use cd to navigate to /var/log, then use pwd to confirm your location, and ls -la to view the contents.
  2. File Operations: Create a directory called practice in your home directory. Inside it, create a file called notes.txt and another directory called backup.
  3. View Logs: Use tail -f /var/log/syslog (or /var/log/messages on some systems) to watch system logs in real-time. Open another terminal and generate activity to see it appear.
  4. System Info: Find out your kernel version with uname -r, disk space with df -h, and memory with free -h.
